## Working note

Smart contracts behave like public system boundaries. They accept signed intent, enforce rules, emit events, and require every connected interface to explain what changed.

## System architecture

The frontend should never treat the wallet as a magic button. It should explain what is being signed, what contract receives it, and what state change is expected.
