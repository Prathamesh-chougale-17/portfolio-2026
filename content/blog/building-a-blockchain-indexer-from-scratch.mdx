## Mission premise

A blockchain indexer is a memory system. It observes chain events, stores them in a queryable shape, and survives replays, retries, and partial failure.

## System architecture

The hardest part is not reading events. The hard part is idempotency. Every event needs a stable identity, every write needs conflict behavior, and every processor needs to tolerate seeing the same signal twice.
